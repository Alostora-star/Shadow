## 2. تعديل نظام الألعاب

### التحسينات الرئيسية:

#### أ. عرض السعر وسؤال المستخدم
عند الضغط على أي باقة (فري فاير أو ببجي)، يظهر للمستخدم:
- 💰 **السعر** بشكل واضح
- ❓ **سؤال: "هل تريد شراء هذه الباقة؟"**
- ✅ زر **"نعم، شراء"** للمتابعة
- ⬅️ زر **"رجوع"** للعودة

**مثال:**
```
💎 فري فاير - 100 الماسة

💰 السعر: $0.90

❓ هل تريد شراء هذه الباقة؟

✅ اضغط على 'نعم، شراء' للمتابعة
⚠️ سيتم خصم المبلغ من محفظتك
```

#### ب. خصم المبلغ أولاً
عند الضغط على "نعم، شراء":
1. ✅ يتم **فحص رصيد المحفظة**
2. ✅ إذا كان الرصيد كافياً، يتم **خصم المبلغ فوراً**
3. ✅ يظهر رسالة تأكيد الخصم مع **الرصيد الجديد**

#### ج. طلب ID اللعبة
بعد خصم المبلغ:
1. ✅ يطلب البوت من المستخدم **إدخال ID اللعبة**
2. ✅ المستخدم يرسل الـ ID في **رسالة نصية**
3. ✅ يتم حفظ الطلب في **قاعدة البيانات**

**مثال الرسالة:**
```
✅ تم خصم المبلغ بنجاح!

💰 المبلغ: $0.90
💳 رصيدك الجديد: $9.10

🎮 الآن، ارسل ID فري فاير الخاص بك:
✏️ اكتب الـ ID وارسله في رسالة نصية
```

## 3. إرسال الطلب للأدمن

### نظام الإشعارات الجديد:

بعد إدخال المستخدم للـ ID، يتم:

#### للمستخدم:
✅ رسالة تأكيد تحتوي على:
- اسم المنتج
- الـ ID المدخل
- المبلغ المدفوع
- الرصيد الجديد
- رسالة "سيتم معالجة طلبك في أقرب وقت"

**مثال:**
```
✅ تم إرسال الطلب بنجاح!

🎮 المنتج: فري فاير - 100 الماسة
🎮 الـ ID: 123456789
💰 المبلغ: $0.90
💳 رصيدك الجديد: $9.10

📦 سيتم معالجة طلبك في أقرب وقت!
```

#### للأدمن:
✅ إشعار فوري يحتوي على:
- اسم المستخدم و ID التليجرام
- اسم المنتج
- **ID اللعبة** (مهم جداً!)
- المبلغ المدفوع
- طلب معالجة الطلب

**مثال:**
```
🔔 طلب شحن جديد!

👤 المستخدم: @username (ID: 123456)
🎮 المنتج: فري فاير - 100 الماسة
🎮 ID اللعبة: 123456789
💰 المبلغ: $0.90

⚠️ يرجى معالجة الطلب في أقرب وقت
```

## 4. حالة الطلب في قاعدة البيانات

- ✅ يتم حفظ الطلب بحالة **"قيد المعالجة"**
- ✅ يتم حفظ **ID اللعبة** في قاعدة البيانات
- ✅ يمكن للمستخدم مشاهدة الطلب في قسم "📋 طلباتي"

## التغييرات التقنية

### ملف `bot.py`:

1. **إضافة حالة محادثة جديدة:**
   ```python
   GAME_ID_INPUT = 8
   ```

2. **تعديل دالة `confirm_game_purchase()`:**
   - فحص الرصيد وخصمه
   - حفظ بيانات الطلب في `context.user_data`
   - طلب ID من المستخدم
   - الانتقال لحالة `GAME_ID_INPUT`

3. **إضافة دالة جديدة `receive_game_id()`:**
   - استقبال ID من المستخدم
   - حفظ الطلب في قاعدة البيانات
   - إرسال تأكيد للمستخدم
   - إرسال إشعار للأدمن مع ID اللعبة
   - إنهاء المحادثة

4. **إضافة `ConversationHandler` للألعاب:**
   ```python
   game_purchase_handler = ConversationHandler(
       entry_points=[
           CallbackQueryHandler(confirm_game_purchase, pattern='^confirm_freefire_'),
           CallbackQueryHandler(confirm_game_purchase, pattern='^confirm_pubg_')
       ],
       states={
           GAME_ID_INPUT: [MessageHandler(filters.TEXT & ~filters.COMMAND, receive_game_id)],
       },
       fallbacks=[CommandHandler("cancel", cancel_deposit)],
       allow_reentry=True
   )
   ```

5. **حذف المعالجات القديمة:**
   - حذف معالجات `confirm_freefire_*` و `confirm_pubg_*` من `handle_callback_query()`
   - لأنها أصبحت entry points في `ConversationHandler`

## سير العمل الكامل

### خطوات الشراء من وجهة نظر المستخدم:

1. 🎮 المستخدم يضغط على "الألعاب"
2. 🔥 يختار اللعبة (فري فاير أو ببجي)
3. 💎 يختار الباقة المطلوبة
4. ❓ يظهر له **السعر** و **سؤال "هل تريد الشراء؟"**
5. ✅ يضغط على "نعم، شراء"
6. 💰 يتم **خصم المبلغ** من محفظته
7. ✏️ يطلب منه **إدخال ID اللعبة**
8. 📤 يرسل الـ ID في رسالة نصية
9. ✅ يحصل على **تأكيد** بإرسال الطلب
10. 📦 ينتظر معالجة الطلب من الأدمن

### خطوات المعالجة من وجهة نظر الأدمن:

1. 🔔 يصل إشعار فوري للأدمن

   - اسم المستخدم
   - المنتج المطلوب
   - **ID اللعبة** (للشحن)
   - المبلغ المدفوع
3. ⚡ الأدمن يقوم بشحن الـ ID
4. ✅ يمكن تحديث حالة الطلب في النظام
